# /loop: PR Review Patrol — 定期评审他人 PR

## Prompt

```
/loop 5,35 * * * * 检查 gitcode-cli/cli 的 open PR，取一个非本人创建的 PR 处理。本次只处理一个，完成后停止，等待下次 Cron 触发，避免上下文膨胀。

前置规则：所有涉及代码的操作必须在独立 git worktree 中执行，严禁污染主工作目录。worktree 用后即删。

1. 门禁检查（构建/UT/Pre-commit/自检/CI/实际命令测试）
2. 缺失 → PR 评论反馈
3. CI 代劳（worktree 中 fetch fork → push GitHub → trigger → 根因分析 → 反馈）
4. 实际命令测试（worktree 中应用改动 → infra-test/* → 反馈）
5. 门禁齐全 + CI 绿 + 命令通过 → 独立多角色评审
6. 已评审无新 commit → 跳过
7. 评审改代码 → worktree 中操作
8. 全部通过 → gc pr merge <N> --yes

必须在 PR 评论区留下每次操作的证据。
禁止: 修改他人代码、跳过评审直接合并、在主目录直接修改代码、仅报 CI 状态不分析根因
```
## 门禁检查清单

| # | 门禁 | 检查 | 缺失反馈 |
|---|------|------|---------|
| 1 | 构建 | go build 结果 | go build -o ./gc ./cmd/gc |
| 2 | UT | go test 结果 | go test ./... |
| 3 | Pre-commit | hooks 通过 | pre-commit run --all-files |
| 4 | 自检 | 9 项模板 | 按 spec 补充 |
| 5 | 实际命令 | infra-test/* | 由 loop 代劳 |
| 6 | CI | Loop 自动补 | — |

## 评论模板

**缺失门禁:** "## 门禁检查（自动）" + 缺失表 + "🤖 Generated by loop patrol"

**CI 反馈:** "## CI 结果（自动）" + Run URL + conclusion + 根因分析 + "🤖 Generated by loop patrol"

**实际命令测试:** "## 实际命令测试（自动）" + 命令 + 结果 + "🤖 Generated by loop patrol"

**独立评审:** "## 独立评审（loop patrol）" + 四角色结论表 + "🤖 Generated by loop patrol"

## .loop/ 维护

每检查一个 PR 后更新 `.loop/deliveries/issue-<N>.md`
