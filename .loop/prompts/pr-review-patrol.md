# /loop: PR Review Patrol — 定期评审他人 PR

## Prompt

```
/loop 30m 检查 gitcode-cli/cli 的 open PR，对每个非本人创建的 PR：
1. 检查 PR 正文/评论中是否包含以下门禁证据：
   - 本地构建: go build 结果
   - UT: go test 结果
   - Pre-commit: hooks 通过记录
   - 作者自检: 9 项模板完整
   - 多角色独立评审: 非作者的 multi-agent 评审结论
   - 远端 CI: CI run URL + 结论

2. 缺失门禁 → 在 PR 评论区列出缺失项 + 操作指引

3. 证据齐全 → 同步分支到 GitHub 镜像仓
   - git push github <branch>
   - unset HTTP_PROXY HTTPS_PROXY http_proxy https_proxy
   - gh workflow run ci.yml --ref <branch>
   - gh run watch <run-id>
   - 在 PR 评论反馈 CI 结果

4. 综合判断:
   - 全部门禁 + CI 全绿 → gc pr merge <N> --yes
   - 有问题 → PR 评论列出具体问题

禁止: 修改他人代码、跳过评审直接合并
```

## 门禁检查清单

| # | 门禁 | 检查内容 | 缺失反馈 |
|---|------|---------|---------|
| 1 | 本地构建 | PR 正文/评论含 go build 成功 | `请运行 go build -o ./gc ./cmd/gc 并在 PR 中反馈` |
| 2 | UT | PR 正文/评论含 go test 全部通过 | `请运行 go test ./... 并在 PR 中反馈` |
| 3 | Pre-commit | PR 正文/评论含 hooks 通过记录 | `请运行 pre-commit run --all-files 并在 PR 中反馈` |
| 4 | 作者自检 | PR 正文/评论含 9 项自检（根因、范围、测试、CI、安全、文档、风险、未覆盖项、主体标识） | `请按 spec/workflows/pr-workflow.md 补充作者自检` |
| 5 | 多角色评审 | PR 评论含非作者的 4 角色评审结论（代码/安全/测试/文档） | `请使用 TeamCreate + Agent 启动多角色评审` |
| 6 | 远端 CI | PR 正文/评论含 CI run URL + 结论 | 由本 loop 自动补充 |
