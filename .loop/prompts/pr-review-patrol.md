# /loop: PR Review Patrol — 定期评审他人 PR

## Prompt

```
/loop 30m 检查 gitcode-cli/cli 的 open PR，对每个非本人创建的 PR：
1. 检查 PR 正文/评论中是否包含以下门禁证据：
   - 本地构建: go build 结果
   - UT: go test 结果
   - Pre-commit: hooks 通过记录
   - 作者自检: 9 项模板完整
   - 多角色独立评审: 非作者的 multi-agent 评审结论
   - 远端 CI: CI run URL + 结论

2. 缺失门禁 → 在 PR 评论区列出缺失项 + 操作指引

3. 证据齐全 → 同步分支到 GitHub 镜像仓
   - git push github <branch>
   - unset HTTP_PROXY HTTPS_PROXY http_proxy https_proxy
   - gh workflow run ci.yml --ref <branch>
   - gh run watch <run-id>
   - 在 PR 评论反馈 CI 结果

4. 综合判断:
   - 全部门禁 + CI 全绿 → gc pr merge <N> --yes
   - 有问题 → PR 评论列出具体问题

5. 每检查一个 PR，更新 .loop/deliveries/issue-<N>.md 记录审查结果

禁止: 修改他人代码、跳过评审直接合并
```

## 门禁检查清单

| # | 门禁 | 检查内容 | 缺失反馈 |
|---|------|---------|---------|
| 1 | 本地构建 | PR 正文/评论含 go build 成功 | `请运行 go build -o ./gc ./cmd/gc 并在 PR 中反馈` |
| 2 | UT | PR 正文/评论含 go test 全部通过 | `请运行 go test ./... 并在 PR 中反馈` |
| 3 | Pre-commit | PR 正文/评论含 hooks 通过记录 | `请运行 pre-commit run --all-files 并在 PR 中反馈` |
| 4 | 作者自检 | PR 正文/评论含 9 项自检模板 | `请按 spec/workflows/pr-workflow.md 补充作者自检` |
| 5 | 多角色评审 | PR 评论含非作者的 4 角色评审结论 | `请使用 TeamCreate + Agent 启动多角色评审` |
| 6 | 远端 CI | PR 正文/评论含 CI run URL + 结论 | 由本 loop 自动补充 |

## PR 评论模板

### 缺失门禁时
```
## 门禁检查（自动）
PR 缺少以下交付证据，请参照 spec/workflows/development-workflow.md 5.3 节补充：
| 门禁 | 状态 | 操作 |
|------|:--:|------|
| 本地构建 | ❌ | go build -o ./gc ./cmd/gc |
🤖 Generated by loop patrol
```

### CI 补充 + 合并
```
## CI 验证（自动）
- Run: <URL> — <conclusion>
## 合并判断
全部通过 ✅ → 自动合并
🤖 Generated by loop patrol
```

## .loop/ 维护

每检查一个 PR 后更新 `.loop/deliveries/issue-<N>.md`:
```markdown
# Delivery Record: Issue #<N>
## Patrol Check
| Date | PR | Gates | CI | Action |
|------|----|-------|----|--------|
| <date> | !<N> | 4/6 missing | — | Commented |
| <date> | !<N> | 6/6 | ✅ | Merged |
```

## 与 full-flow.md 的分工

| 场景 | 处理者 |
|------|--------|
| 本人创建的 PR | `full-flow.md` |
| 他人创建的 PR | `pr-review-patrol.md` |
| risk/high 待确认 | 两条线都暂停 |
