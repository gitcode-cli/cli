# Human Approval Policy

## 目标

定义 Loop Engineering 中必须保留人工确认或独立主体确认的节点。

## 必须人工确认

- 规范未覆盖的行为
- 规则冲突
- 高风险变更范围扩大
- 远端仓库创建或删除
- 合并 PR
- 发布 release
- 将一次性经验沉淀为长期规范
- 任何涉及凭证、权限、删除、覆盖、迁移的操作

## 独立评审要求

- 作者自检不能替代独立评审
- 独立评审必须能看到 diff、测试、CI、风险和未覆盖项
- review 结论应写回 GitCode PR

## AI 可自动执行

在规则已覆盖且不越过人工确认点时，AI 可以：

- 读取 issue / PR / CI 状态
- 生成计划和证据草稿
- 修改非 main 分支
- 运行本地验证
- 创建 PR
- 回写自检、CI evidence 和归档建议

AI 不得自动 merge、发布或绕过人工确认点。
